# 梨花祭 入場システム 実証実験計画書

**作成日：** 2026年5月23日  
**発表予定：** 2026年6月12日（職員会議）

---

## 1. 目的

梨花祭の入場管理システムは開発が完了しているが、実際の運用経験がない。  
本番（梨花祭）で安定稼働させるために、**朝の登校時間帯に生徒自身がチェックインを行う実証実験**を実施し、以下を検証する。

- システムが実際の運用に耐えられるか（安定性・速度）
- スタッフがQRスキャン操作をスムーズに行えるか（操作性）
- 問題点・改善点を本番前に洗い出せるか（事前改善）

## 3. 実験フロー

### 事前準備（実験〜3日前まで）

```
管理者が参加生徒全員分の「実験用ゲスト枠」を1枠追加割り当て
↓
各生徒へ通知（LHRや連絡プリント等）
「/student にアクセスして招待リンクを1件発行し、自分のスマホで開いてQRを保存してください」
↓
生徒：学校メールアドレスでOTPログイン → 招待リンクを生成 → 自分でリンクにアクセス → QRコードをスクリーンショット保存
```

### 実験当日

```
スタッフが /admin/verify にアクセス（スキャン画面を起動）
↓
登校してきた生徒がスマホのQRコードを提示
↓
スタッフがQRをスキャン
↓
システムが「チェックイン済み」を記録（時刻も自動記録）
↓
（次の生徒へ）
```

### 実験後（データ分析）

```
管理者ダッシュボードで登校時刻・チェックイン率を確認
↓
スタッフへのヒアリング（操作感・困った点）
↓
生徒へのアンケート（QR保存の難易度、手順のわかりやすさ）
↓
梨花祭本番に向けた改善点リストアップ
```

## 4. 評価指標

| 指標 | 目標値 | 計測方法 |
|------|--------|----------|
| QRスキャン成功率 | 95%以上 | システム記録 |
| 1人あたりの処理時間 | 5秒以内 | ストップウォッチ計測 |
| システム障害件数 | 0件 | スタッフ記録 |
| 生徒のQRコード準備率 | 90%以上 | チェックイン記録 |
| スタッフ操作習熟度 | 練習5分以内で一人で操作可能 | 事前訓練記録 |

---

## 5. リスクと対策

| リスク | 対策 |
|--------|------|
| QRコードが読み取れない | 生徒ID等による手動入力バックアップ手順を用意 |
| スマホを忘れた・QR未保存 | その場でOTPログインしてQR再取得する手順を準備 |
| ネットワーク障害 | モバイルWi-Fiルーターを予備で用意 |
| 生徒が手順を理解できない | 前日にLHRでデモ・プリント配布 |
| スタッフが操作できない | 実験前日に15分の練習セッションを実施 |

## 7. 梨花祭との違い（実験の位置づけ）

梨花祭本番との違いは「ゲストが外部の人かどうか」だけで、**システムの操作・フローはまったく同じ**。

| 項目 | 実証実験 | 梨花祭本番 |
|------|----------|-----------|
| チェックイン対象 | 生徒自身 | 外部ゲスト |
| QR取得方法 | 生徒が自分の招待枠を自分で使う | 生徒がゲストに招待リンクを送る |
| 規模 | 一部クラス（数十名） | 全校（数百名以上） |
| システム | 同一 | 同一 |

実証実験で得られた**操作手順・改善点・スタッフ習熟度**はそのまま本番に活かせる。

## 9. 期待される効果

1. **梨花祭本番前の「ドライラン」** として機能し、不具合を安全な環境で発見できる
2. スタッフが操作に慣れることで本番当日の混乱を防ぐ
3. 生徒がシステムに慣れ、梨花祭でスムーズにゲストを招待できるようになる
4. 実際の運用データから、混雑ピーク時間・処理速度などを把握できる
