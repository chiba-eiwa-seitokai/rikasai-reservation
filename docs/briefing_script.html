② システム・当日の概要（0:05〜　10分）
まずシステムの仕組みを簡単に説明します。

梨花祭では、来場者（招待者・保護者）は事前に生徒から招待リンクを受け取り、そこからQRコードを取得してスマートフォンに保存してきます。
当日みなさんの仕事は、正門前のテントでそのQRコードをスキャンして入場確認をすることです。

配布資料の1ページ目を見てください。テントは2種類あります。「招待者テント」と「保護者テント」です。
みなさんの担当テントは当日または前日に別途お知らせします。
「招待者・保護者の違い」「担当テントは後日連絡」の2点が伝わっているか確認。
受付時間は9時から14時です。9時前には集合してテントの設置と動作確認をしてもらいます。
具体的な集合時刻は当日資料で確認してください。
③ 操作手順の実演（0:15〜　10分）
端末1台を手に持ち、全員から見える位置に立つ。スキャン画面（/admin/verify）を開く。
では実際の操作を見てもらいます。配布資料1ページ目の「操作手順」と照らし合わせながら見てください。

まず、ブラウザでスキャン画面を開きます。URLは「〇〇〇〇/admin/verify」です。
こうするとカメラが起動してスキャン待機状態になります。
事前に準備したテスト用QRコードをスキャンして「入場OK」画面を全員に見せる。
QRコードをカメラに向けると…こうやって名前が表示されます。これが「入場OK」の状態です。
この画面が出たらゲストを昇降口へ案内してください。
同じQRを再度スキャンして「入場済み」エラーを見せる。
同じQRをもう一度スキャンするとこうなります。「入場済み」と表示されます。
これは二重入場を防ぐためです。このエラーが出たら生徒会担当に連絡してください。

QRが読み取れないとき、またはゲストがQRを持っていないときは、自分で判断せず担当の先生のところへゲストを案内してください。
手書き記録での対応は先生だけが行います。実行委員はしません。詳しくは資料2ページ目を確認してください。
「入場OK画面」と「エラー画面」の両方を見せたか確認。質問があれば受ける。

④ 実機練習（0:25〜　20分）
参加者を端末3台に対応した3グループに分ける。シフトの人数に応じてグループ規模を調整する。各グループに端末1台を渡す。
では実際に体験してもらいます。3グループに分かれてください。
各グループに端末を1台ずつ渡します。グループ内で順番に練習してください。

練習の流れはこうです。

実機練習の進め方（各グループ）

  1. 端末でスキャン画面（/admin/verify）を開く
  2. 隣の人のスマートフォンにテスト用QRを表示してもらい、スキャンする
  3. 「入場OK」が表示されたら成功。次の人に交代
  4. 全員1回ずつ体験したら、エラー画面も確認する（同じQRを2回スキャン）
練習中は各グループを回り、操作に詰まっている人をサポートする。残り5分で全体に声かけ。
全員が最低1回スキャンを体験できたか確認してから次へ進む。
